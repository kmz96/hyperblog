Vamos a explorar de nuevo esto

Utilizaremos git branch para crear una nueva rama llamada "cabecera" posteriormente nos movemos a cabecera con git checkout "cabecera"

Vamos a hacer un resumen
1-git add = agrega los archivos a staging
2-git commit -m "Comentario"= Pasa de staging a repositorio local
Nota: git commit -am "Comentario" = paso 1+2
3-git status = dice que archivos estan en staging o sin rastrear
4- git log = devuelve el historial de cambios a cada rama
5-git show = muestra que cambios hubo en cada commit
6-git branch NombreRama = crea una nueva rama
7-git checkout rama = cambia a la rama indicada
